(automated server startup program)
SECOND, if you double click "START_HERE.bat", a terminal will auto-opened and you will be guided through the automated dependency installation process.

The automated server startup program will clean, restore, build and start the server locally on port 5093.
 - If starting on port 5093 fails, the program will try each port between 5093-5100. This is only the case with the automated server startup, not if you're proceeding manually.
 - If all attempts fail, the program will print this error message:
"Could not find a free port from 5093 to 5100.
Close other running copies of the app, or run manually with:
dotnet run --project ... --urls http://127.0.0.1:5093"
(Only change "5093" to another port number if you know what you're doing.)

If you would rather do this manually, DO NOT click "START_HERE.bat".



1. Make sure your system has the .NET 10 SDK installed.
   On Windows, you can install it with:
   winget install Microsoft.DotNet.SDK.10

2. To manually start the server, run:

 - cd "FULL_PATH_TO\MovieWatchlistTracker"
 - dotnet clean .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
 - dotnet restore .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
 - dotnet restore .\tests\MovieWatchlistTracker.Tests\MovieWatchlistTracker.Tests.csproj
 - dotnet build .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
 - dotnet run --project .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj --urls http://127.0.0.1:5093

3. Once the server is running, open this URL in your preferred browser:

   http://127.0.0.1:5093

   If you changed the port number in the command above (5093), use your port number instead.

4. To shut down the server, press Ctrl+C in the terminal running the server.
