FIRST, if you haven't already, place this app as far up in your filesystem (avoid nested directories) as this app contains file with very long names. If attempting to unzip this app under such conditions, you may incur "Error 0x80010135: Path too long".

SECOND, if you double click "START_HERE.bat", you will be guided through the automated dependency installation process.

If you would rather do this manually, DO NOT click "START_HERE.bat".

1. Make sure your system already has either .NET-SDK-10 (to edit and run app) or ASSP.NET10 (to only run app).
 - Cleanest installation methods for Windows:
  - In a terminal, enter either "winget install Microsoft.DotNet.SDK.10" or "winget install Microsoft.DotNet.AspNetCore.10" (without quotation marks). 

2. To manually start server, run the following commands:
  2a. cd "NEW_LOCATION\MovieWatchlistTracker_proj\MovieWatchlistTracker"
  2b. dotnet clean .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
  2c. dotnet restore .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
  2d. dotnet restore .\tests\MovieWatchlistTracker.Tests\MovieWatchlistTracker.Tests.csproj
  2e. dotnet build .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj
  2f. dotnet run --project .\src\MovieWatchlistTracker.Web\MovieWatchlistTracker.Web.csproj --urls http://127.0.0.1:5093

This will clean, restore, build and start the server locally on port 5093.
 - If starting on port 5093 fails, the program will try each port between 5093-5100.
 - If all attempts fail, the program will print this error message:
"Could not find a free port from 5093 to 5100.
Close other running copies of the app, or run manually with:
dotnet run --project ... --urls http://127.0.0.1:YOUR_PORT" (EDIT "YOUR_PORT" TO YOUR PREFERRED PORT NUMBER)

3. Once the server is running, enter "http://127.0.0.1:YOUR_PORT" into your preferred browser.

4. To shut down the server, enter CTRL+C into the terminal that is running the server.
